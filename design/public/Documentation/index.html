Introduction

Youxi is a clean, responsive and retina ready HTML5 theme build exclusively for sale at ThemeForest.

This template is built using HTML5 and CSS3 utilizing the last web techniques and technology. It has a professional, clean design suitable for any business, personal or portfolio websites.

Packed with tons of widgets, elements and styles all coded using the last standard with the minimum required external images as possible, this template has been tested on various devices and browsers, assuring an optimal user experience on almost every platform.

Customization of the template is also very easy, since every element is coded in a modular way without any dependencies on each other. Color customization also, can be done quickly with the provided theme specific LESS file.

Support Request

Browser support is targeted at IE8+, Mozilla Firefox, Google Chrome, Apple Safari, Opera, all Android and iOS mobile browsers, for browsers other than those support is not available.

Besides the @font-face and icon stlyes, these files are mandatory for each of your pages to get all elements working

  1. assets/css/normalize.css

  6. assets/css/shortcodes.css

  7. assets/css/elements.css

Building the base HTML

In order to understand how this template works, the basic HTML structure will be covered and explained here. By reading this section, you should be able to build an empty, usable HTML page as the base of your website.

Building the base HTML structure is pretty straightforward, we'll now go through a complete process on creating a complete page that includes the minimal required JavaScript and CSS files.

<![endif]-->

<title>Youxi - Responsive HTML5 Business Theme</title>

</head>

Like how usually one writes a HTML page, we always need the proper doctype and HTML opening tag. Here we use the HTML5 doctype, in order to make use of the latest web technology and standards.
The HTML5 doctype assures that all features related to HTML5 such as the HTML data-* attributes are working properly.

Following the HTML5 doctype, we'll have our HTML opening tag surrounded by IE conditional tags to accomodate the older IE browsers (IE < 10).
This is the correct way to style webpages on older IE as written by Paul Irish (http://paulirish.com/2008/conditional-stylesheets-vs-css-hacks-answer-neither/).
Using this technique, we don't need to have our webpage cluttered with IE conditional tags, and styling elements can just be done by prefixing the IE specific styles with the class added to the HTML element.

Following the HTML opening tag, we'll have our <head> element. It will contain the required metatags, stylesheets and HTML5 shim as can be seen on the code snippet above.
The metatags defined above are required for the responsive layout on mobile devices and thus can't be ommited.
The stylesheets was already explained in the previous section, so after that we'll have the HTML5 shim. This one is required to get IE to display HTML5 elements correctly.

Make sure that your webpages contain those lines to get everything working correctly.

Skeleton; A Beautiful Boilerplate for Responsive, Mobile-Friendly Development

Skeleton is a small collection of CSS files that can help you rapidly develop sites that look beautiful at any size, be it a 17" laptop screen or an iPhone. Skeleton is built on three core principles:

  • Responsive Grid Down To Mobile
    Skeleton has a familiar, lightweight 960 grid as its base, but elegantly scales down to downsized browser windows, tablets, mobile phones (in landscape and portrait).
  • Fast to Start
    Skeleton is a tool for rapid development. Get started fast with CSS best practices, a well-structured grid that makes mobile consideration easy, an organized file structure and super basic UI elements like lightly styled forms, buttons, tabs and more.
  • Style Agnostic
    Skeleton is not a UI framework. It's a development kit that provides the most basic styles as a foundation, but is ready to adopt whatever your design or style is.

In short, skeleton is a CSS framework to define a responsive, mobile friendly grid system that is utilized by this template as the main content structure.

Below is an explanation about skeleton and its usage taken from skeleton's official website (http://www.getskeleton.com/)

The Grids

Skeleton's base grid is a variation of the 960 grid system. The syntax is simple and it's effective cross browser, but the awesome part is that it also has the flexibility to go mobile like a champ.

<!-- The container is a centered 960px -->
<div class="container">

<!-- Give column value in word form (one, two..., twelve) -->
<div class="sixteen columns">
    <h1>Full Width Column</h1>
</div>

<!-- Sweet nested columns cleared with a clearfix class -->
<div class="six columns clearfix">
    <!-- In nested columns give the first column a class of alpha
    and the second a class of omega -->
    <div class="three columns alpha">...</div>
    <div class="three columns omega">...</div>
</div>

<!-- Sweet nested columns cleared by wrapping a .row -->
<div class="five columns">
    <div class="row">
        <div class="three columns alpha">...</div>
        <div class="two columns omega">...</div>
    </div>
</div>

<!-- Sweet nested columns cleared by <div class="clear"></div> -->
<div class="five columns">
    <div class="three columns alpha">...</div>
    <div class="two columns omega">...</div>
    <div class="clear"></div>
</div>

<!-- Can push over by columns -->
<div class="five columns offset-by-one"></div>

</div>

Media Queries

Skeleton uses a lot of media queries to serve the scalable grid, but also for the convenience of styling your site on different size screens. Skeleton's media queries are almost exclusively targeted at max and min widths rather than device sizes or orientations. The advantage of this is browsers and future mobile devices that don't map to exact set dimensions will still benefit from the styles. That being said, all of the queries were written to be optimal on Apple iOS devices. The built in media queries include:

  • Smaller than 960: Smaller than the standard base grid
  • Tablet Portrait: Between 768px and 959px
  • All Mobile Sizes: Less than 767px
  • Just Mobile Landscape: Between 480px and 767px
  • Just Mobile Portrait: Less than 479px

Using AJAX Contact Form

This template includes a working contact form. It can help your visitors to send message to a specified email address without the need of additional scripts. It utilizes jQuery Form, jQuery Validation plugin on the client side, SwiftMailer and PHP-Validation library on the server side.
Just put the required PHP files on your PHP server, configure your form, alter the PHP configurations and you can start receiving emails from your visitor.

Configuring The PHP Variables

The /PHP folder that you can find in this template's root folder contains several files and folder.
You don't need to configure or change anything besides these two files: mail-content.php and contact.php

  • mail-content.php
    This file contains the message that you'll receive from your visitors. Edit this file, and print the variables that matches your contact form fields name attribute.
    <?php echo $address ?>
    This prints out the $address variable, which matches your contact form field with the name="ContactForm[address]" attribute.
  • contact.php
    This is the script that receives the form's submission. It basically validates the form, and if the form is valid it will call the necessary functions to send your visitor's email then responds the request with the submission result/message.

Validating The Form Client Side

Form validation on the client side is done by using jQuery Validation plugin. To use this plugin, just specify the classnames on your form fields, and the form will be validated without further configuration.
Documentation of the form validation plugin can be read here: http://docs.jquery.com/Plugins/Validation.

Validating The Form Server Side

In the demo PHP file, you can see a set of validation rules being applied to the contact form data. If your contact form has a different structure than the provided one, you should also change the validation rules in the contact.php file.
Server side validation is done by using PHP-Validation library and for more info how to configure the validation of your form fields, you should read the docs of the validation library here: https://github.com/Dachande663/PHP-Validation
Make sure that you validate your form fields to prevent unwanted data sent to your email. 

$validator
    ->set_label('name', 'Name')
    ->set_label('email', 'Email')
    ->set_label('subject', 'Subject')
    ->set_label('comment', 'Comment')
    ->add_filter('name', 'trim')
    ->add_filter('email', 'trim')
    ->add_filter('email', 'strtolower')
    ->add_filter('subject', 'trim')
    ->add_rule('name', new Rule\NotEmpty())
    ->add_rule('email', new Rule\NotEmpty())
    ->add_rule('email', new Rule\Email())
    ->add_rule('subject', new Rule\NotEmpty())
    ->add_rule('comment', new Rule\NotEmpty());

Changing Color Scheme

This template's main color can be changed easily by compiling the provided LESS file which is located in /themer/less/.
Just change the variable values on top of this file, compile the LESS file and include it on your page.

Actually, everything can be done on the demo page easily, just use the Themer on the left side of the page and hit the 'Get CSS' button. This will compile the LESS file with the selected color, and trigger a modal window that contains the generated CSS file.
